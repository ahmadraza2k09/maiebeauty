# Vercel Deployment Guide

This document explains how to deploy your maiebeauty TanStack Start project to Vercel.

## Pre-Deployment Checklist

- [ ] Git repository initialized and all changes committed
- [ ] Environment variables configured (see `.env.example`)
- [ ] Build command verified: `npm run build`
- [ ] All tests passing (if applicable)
- [ ] `vercel.json` configuration in place

## Deployment Steps

### 1. Install Vercel CLI (Optional)

```bash
npm install -g vercel
```

### 2. Deploy to Vercel

#### Option A: Using Vercel Dashboard (Recommended for first-time)

1. Go to [vercel.com](https://vercel.com)
2. Sign in or create an account
3. Click "Add New" → "Project"
4. Select "Import Git Repository"
5. Authorize and select this repository
6. Configure project settings:
   - **Framework Preset**: Other
   - **Build Command**: `npm run build`
   - **Output Directory**: `.output`
   - **Install Command**: `npm install`
7. Add environment variables (if any)
8. Click "Deploy"

#### Option B: Using Vercel CLI

```bash
# Login to Vercel
vercel login

# Deploy
vercel --prod

# Or for preview deployment
vercel
```

### 3. Configure Environment Variables

In Vercel Dashboard:
1. Go to your project settings
2. Navigate to "Environment Variables"
3. Add variables from `.env.example` and update values for production

Common variables to configure:
- `VITE_API_URL`: Your production API endpoint
- `VITE_ENV`: Set to "production"
- `VITE_ENABLE_ANALYTICS`: Set to "true"

### 4. Build Configuration

The `vercel.json` file includes:
- **buildCommand**: Runs Vite build with Nitro for Node.js runtime
- **outputDirectory**: `.output` - Generated by Nitro build process
- **nodeVersion**: 20.x
- **headers**: Caching policies for assets and API routes
- **environment**: Production environment settings

### 5. Verify Deployment

After deployment:
1. Check the deployment URL
2. Test all routes and functionality
3. Verify API connections
4. Check console for any errors
5. Test responsive design across devices

## Troubleshooting

### Build Fails with "Cannot find module"

- Ensure all dependencies are listed in `package.json`
- Run `npm install` locally to verify
- Check that imports use correct paths with `@/` alias

### Assets not loading

- Verify asset paths in Vercel deployment logs
- Check that `public/` files are being served correctly
- Ensure asset imports are using the correct relative paths

### Environment Variables Not Working

- Verify variable names match in code (case-sensitive)
- Ensure variables are prefixed with `VITE_` for client-side variables
- Rebuild after adding/changing environment variables

### Server Routes Returning 404

- Verify API routes exist in the output
- Check that server functions are properly exported
- Ensure build completed successfully

## Continuous Deployment

For automatic deployments on git push:
1. Link your repository in Vercel Dashboard
2. Set deployment branch (typically `main`)
3. Configure automatic deployments on push
4. Each push will automatically build and deploy

## Performance Optimization

### Image Optimization
- Use the asset loader for images (`.asset.json` files)
- Consider using Vercel Image Optimization

### Caching Strategy
- Static assets cached for 1 year (`cache-control: immutable`)
- API routes cached for 1 hour
- Adjust in `vercel.json` as needed

### Function Configuration
- Memory: 1024 MB (adjust if needed)
- Max Duration: 60 seconds for functions

## Rollback

To rollback to a previous deployment:
1. Go to Vercel Dashboard
2. Navigate to "Deployments"
3. Find the desired previous deployment
4. Click the three dots menu
5. Select "Promote to Production"
